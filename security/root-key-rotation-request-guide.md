## Purpose
This guide explains how to request a root key rotation through approved support channels.

> **Note:** Sanitized portfolio sample. Workflow clarity preserved without revealing internal systems.

---

## Prerequisites
Before submitting a request, ensure you have:
- Administrative authorization
- A valid reason for rotation
- Availability for a maintenance window (if required)

---

## Request Process

### 1) Submit a Request
Provide:
- Environment identifier
- Authorized approver
- Reason for rotation
- Preferred execution window

### 2) Validation
The request is reviewed to confirm authorization and completeness.

### 3) Execution & Confirmation
Once approved:
- The rotation is executed securely
- System stability is verified
- Confirmation is sent

---

## Security Notes
- Root credentials are never shared over email or chat
- Unauthorized requests are rejected
